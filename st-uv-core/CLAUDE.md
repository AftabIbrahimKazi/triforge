# st-uv-core — Claude Code Guide

CPU-side UV unwrapping algorithms for Three.js BufferGeometry.
Mirrors Blender's UV unwrap operations: same names, same parameters.
Root CLAUDE.md rules always take precedence.

---

## Key Invariants

### Every unwrapper must have:
- `readonly unwrapType: string` — unique identifier matching Blender's name
- `parameters: Record<string, number>` — all scalar inputs as plain numbers (GSAP-safe)
- `apply(geometry: BufferGeometry): BufferGeometry` — returns a new geometry with updated UV attribute
- `enabled: boolean` — when false, apply() returns a geometry clone with no UV changes

### Non-destructive
Never mutate the input geometry. Always `geometry.clone()` and modify the clone.

### UV attribute target
Write to the `'uv'` attribute (not `'uv2'` or `'uv3'`).
Use `new BufferAttribute(uv, 2)` and `result.setAttribute('uv', ...)`.

### Both indexed and non-indexed geometry
`buildMeshGraph()` handles both. The projection unwrappers (Cube, Cylinder, Sphere) operate
per-vertex and work with any geometry. LSCM/ABF require indexed geometry — they call
`buildMeshGraph()` which welds non-indexed vertices.

## Algorithm Details

### Projection unwrappers (CubeProjection, CylinderProjection, SphereProjection)
Direct mathematical mapping, no graph construction needed.
O(n) in vertex count.

### SmartUVProject
1. Compute per-face normals
2. Greedy cluster faces by normal similarity within `angleLimit` degrees
3. Project each cluster onto its average normal plane (orthographic)
4. Shelf-pack all islands into [0,1]²
O(F²) worst case for clustering (F = face count), O(F log F) for packing.

### ConformalLSCM (cotangent Laplacian harmonic map)
1. Build mesh graph from `buildMeshGraph()`
2. Find 2 pinned vertices (boundary endpoints, or farthest pair for closed meshes)
3. Build cotangent-weighted Laplacian system: for each free vertex sum_j w_ij*(u_j - u_i) = 0
4. Solve u and v independently with conjugate gradient
5. Remap result to [0,1]²
Cotangent weights = cot(α) + cot(β) per edge, where α,β are opposite angles.
Weights are clamped to ≥ 0 to handle obtuse triangles.

### AngleBasedABF (mean-value coordinates)
Same structure as LSCM but uses mean-value weights:
w_ij = tan(θ/2) / |pi - pj| (always positive, better for obtuse triangles).
Approximates ABF's angle-minimization behaviour.

### PackIslands
1. BFS to find connected UV islands
2. Compute 3D area per island
3. Scale islands proportional to √(3D area) for consistent texel density
4. Shelf-pack into [0,1]²

### AverageIslandScale
Computes texel density ratio (√3D_area / √UV_area) per island, scales all to the median.

### MarkSeams
Stores seam edges in `geometry.userData.seams: Array<{a:number, b:number}>`.
Does not modify geometry positions or UVs — metadata only.
LSCM/ABF do not yet read seams (future: use seams to cut closed meshes).

## Conjugate Gradient Solver (linearSolver.ts)

`cgSolve(A, b, x0?, maxIter?, tol?)` — solves A*x = b for symmetric PSD A.
Sparse matrix: `SparseMatrix = Map<number, number>[]` (per-row adjacency).
`sparseAdd(A, i, j, val)` — accumulate value into sparse entry (i, j).
`sparseZero(n)` — create n×n zero sparse matrix.

Typical convergence: 50–200 iterations for UV parameterization.

## Adding a New Unwrapper

1. Create `src/unwrappers/MyUnwrapper.ts`
2. Extend `BaseUnwrapper`
3. Name to match Blender exactly
4. All numeric inputs in `parameters` object
5. Return a NEW BufferGeometry — never mutate input
6. Export class and option type from `src/index.ts`
7. Add tests in `test/run-tests.js` and rebuild: `npm run build && npm test`
8. Update `TUTORIAL.md` in this package and the root `TUTORIAL.md`
9. Update root `BACKLOG.md`

## Performance Notes

- LSCM/ABF: O(n * iter) where iter ≈ 50–200. Fine for meshes up to ~50k vertices.
- SmartUVProject clustering is O(F * G) where G = number of islands. With typical meshes
  G is small (~10s of islands), so this is effectively O(F).
- PackIslands and AverageIslandScale are O(V + T) where T = triangle count.
- All operations are CPU-only. Call once per parameter change, not inside the render loop.
