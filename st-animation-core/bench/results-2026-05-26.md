# st-animation-core benchmark — 2026-05-26

Node.js 24, Windows 11.

| Operation | µs/op | Notes |
|---|---|---|
| ShapeKeyMesh.update (1000 verts, 3 keys) | 136 | CPU blending — call once per frame; GPU morph targets faster for high vertex counts |
| Armature.update (20 bones, chain) | 8.8 | Full FK chain recompute |
| SkinBinding.apply (500 verts, 20 bones) | 72 | CPU skinning — acceptable for < 1k verts |
| NLAEditor.update (4 tracks, 2 strips) | 0.7 | Negligible overhead |

## Notes

- Shape key blend at 136 µs for 1k verts: within budget for characters (typical < 10k verts = < 1.4 ms).
- Armature at 8.8 µs for 20 bones: suitable for real-time characters.
- CPU skinning at 72 µs for 500 verts: use THREE.SkinnedMesh + GPU skinning for > 2k verts.
- NLAEditor at 0.7 µs: negligible; supports many tracks without concern.
