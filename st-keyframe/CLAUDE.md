# st-keyframe — Claude Code Guide

CPU-side keyframe animation driver for the Three.js ecosystem.
Drives any `parameters` plain object on any ecosystem class.
Mirrors Blender's F-Curve / NLA system: same concepts, same naming.
Root CLAUDE.md rules always take precedence.

---

## Package Structure

```
src/
  core/
    Keyframe.ts          — Keyframe interface + interpolateKeyframes()
    KeyframeTrack.ts     — Drives one numeric property on one target object
    AnimationClip.ts     — Named collection of KeyframeTracks (Blender: Action)
    AnimationMixer.ts    — Drives clips forward in time (Blender: NLA)
    KeyframeBuilder.ts   — buildClip() fluent helper
  easing/
    easings.ts           — All easing functions + Blender aliases
  index.ts
test/
  run-tests.js           — 39 Node.js tests (no framework, imports from dist/)
bench/
  run-bench.js
  results-YYYY-MM-DD.md
dist/                    — compiled output (generated by tsc)
```

## Key Invariants

### Parameters object contract
Every ecosystem class already has `parameters: Record<string, number>`.
`KeyframeTrack` accepts any `Record<string, number>` as its target — this is the only coupling.
No package-specific imports. No type narrowing. Duck-typed.

### No Three.js dependency
st-keyframe has NO peer dependency on three. It is pure TypeScript.
It drives `parameters` objects, not Three.js internals.
Three.js calls (mesh.rotation.x = ...) happen in user code after `clip.evaluate()`.

### Zero allocation in the hot path
`KeyframeTrack.evaluate()` is designed for render-loop use.
- Binary search: no array allocations
- Writes directly to `target[property]`
- No closures created per evaluate call

### Easing applies from the current keyframe to the next
`kf.easing` is the easing function used from `kf` to the *next* keyframe.
The last keyframe's easing is ignored (nothing after it).
This matches Blender's F-Curve handle-right behaviour.

## Blender Mapping

| st-keyframe | Blender |
|---|---|
| `Keyframe` | F-Curve keyframe point |
| `KeyframeTrack` | F-Curve (one channel = one property) |
| `AnimationClip` | Action |
| `AnimationMixer` | NLA Editor / playback engine |
| `WrapMode` | NLA strip "Extrapolation" |
| `easing` on keyframe | F-Curve interpolation mode |
| `Easings.LINEAR` | F-Curve Linear interpolation |
| `Easings.CONSTANT` | F-Curve Constant interpolation |
| `Easings.BEZIER` | F-Curve Bezier (approximated as easeInOutCubic) |

## Adding a New Easing

Add to `src/easing/easings.ts`:
1. `export const myEasing: EasingFn = (t) => ...`
2. Add to the `Easings` registry at the bottom
3. Export from `src/index.ts`
4. Add a test verifying `myEasing(0) === 0` and `myEasing(1) === 1`

## Performance Notes

- `evaluate` at 0.16–0.39 µs/op: animating 100 parameters per frame costs ~40 µs — well inside 16 ms.
- `buildClip` at ~26 µs: one-time construction cost, not per-frame.
- Binary search in evaluate: O(log n) — 50 keyframes is as fast as 2 keyframes in practice.
- All operations are CPU-only and allocation-free in the hot path.
