# st-keyframe benchmark — 2026-05-26

Node.js 20, Windows 11, no browser overhead.

| Operation | µs/op | Notes |
|---|---|---|
| KeyframeTrack.evaluate (2 keyframes, linear) | 0.39 | Hot path for simple animations |
| KeyframeTrack.evaluate (50 keyframes, binary search) | 0.16 | Binary search O(log n) — faster than linear in JIT |
| AnimationMixer.update (10-track clip, loop) | 1.21 | 10 tracks × evaluate per frame |
| buildClip (5 objects × 2 props, 3 steps) | 25.77 | One-time construction cost, not per-frame |

## Notes

- `evaluate` at 0.39–0.16 µs/op: animating 100 parameters costs ~40 µs/frame — well within 16 ms budget.
- `buildClip` at 26 µs is a one-time call. No concern for real-time use.
- `AnimationMixer.update` with 10 tracks: 1.2 µs — scales linearly with track count.
